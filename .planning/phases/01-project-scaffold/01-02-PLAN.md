<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local to GitHub | Source code pushed to public repository |
| GitHub to Vercel | Vercel pulls and builds from GitHub repo |
| Vercel CDN to end user | Static assets served to browsers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | Public GitHub repo | accept | No secrets in codebase. App is a client-side calculator with no sensitive data. Public repo is intentional. |
| T-01-06 | Information Disclosure | Vercel deployment tokens | mitigate | Vercel CLI uses locally stored auth token (~/.config/vercel). Token not committed to repo. No VERCEL_TOKEN in .env or CI config. |
| T-01-07 | Tampering | Vercel build pipeline | accept | Vercel runs npm run build in isolated container. Build output is static files only. No server-side code executes. |
| T-01-SC | Tampering | npm installs during Vercel build | mitigate | package-lock.json committed; Vercel uses `npm ci` which installs exact versions from lockfile, preventing supply chain drift between local and deployed builds. |
</threat_model>
